Writing Posts

Sunday January 1, 2023

This post has some general information about how to write a post.

Meta Data

The material at the top of one of these files, on lines that start with two periods, is “meta data” about the post.

Title

The title is what the web page will display. There are no restrictions on this.

Date

The date will be displayed below the title. Enter it here in year-month-day form: yyyy-mm-dd. You will be able to see posts with a date in the future on the staging version of the website is visible only on this computer. The website visible to the outside world will not display them until the date is current or in the past.

Using a date in the near future lets you schedule in advance when a post goes live. In the meantime, you can still preview it on this machine. A post like this one with a date in the far future lets us have posts visible only to you and me. They give you a way to experiment and to save information for reference.

The slug

The slug will end up being both the name of this file and part of the url for finding the webpage. Because it is part of the url, it must not have any spaces or any capital letters.

You can save this file with any name you want, e.g. newpost.md. Overnight, a program that works in the background will use the slug to rename it as 1_writing_posts.md. The leading tilde shows that this is a post in the future. Together, the numbers and the tilde help organize the posts so that they show up in the panel on the left in the order implied by the dates in the meta data. It is ok for these file names to change. What should not change after a post goes live, is the slug.

As you will see on the local staging website, the url for this post will be

sitename/posts/writing_posts

On the staging site, “sitename” will be the ugly string “127.0.0.1:7600” On the web, it will be: "carolineweber.net"

Search engines use the words in the url to guide people to pages, so it is good to have some descriptive terms in the slug.

Markdown Preview

If you use the key combination Control + Shift + M, a preview window will open on the right. It gives you a hint about how your text will be formatted on the webpage.

Open up the preview window. For example, notice how the “> ” before the URL three paragraphs back sets it off as a block quote. As you know, inclosing a phrase with a pair of single asterisk gives you italics: text in an italic face. Doubled asterisks gives bold face: Text that will display in boldface.

For each post, there will be a short excerpt that shows on any listing of posts, e.g. on the home page or the blog page. You can put anyplace where you want the short excerpt from the post to end. You can put it even in the middle of a sentence. In the full post, this tag will be replaced by a single space between the two words, in this case, between “put” and "anyplace"

The excerpt will end with a link to the full post and any text you want. Usually, it is "Read more ..."
